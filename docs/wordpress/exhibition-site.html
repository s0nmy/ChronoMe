課題

個人開発・学習・副業では、複数作業の時間配分が曖昧になりやすい。

解決

プロジェクト・タグ・レポートで、作業の量と偏りを視覚的に把握できる。

01 企画

チーム向け勤怠管理ではなく、個人が毎日使い続けられる軽さに焦点を絞りました。 記録、分類、振り返りを一つの流れにし、時間管理の習慣化を狙っています。

02 コーディング技術

フロントエンドは React + TypeScript + Vite、API通信は TanStack Query。 バックエンドは Go + Chi + GORM を用い、Clean Architectureで責務を分離しています。

React TypeScript Go Chi GORM

03 インフラ技術

開発初期は SQLite で素早く検証し、DSN切替で PostgreSQL へ移行できる構成にしています。 Cloud Run、Nginx、Docker、Cloud SQL を想定したデプロイ設計も整理済みです。

04 UI／UX、デザイン

タイマー、タイムライン、日次・週次・月次レポートを中心に、入力の速さと振り返りやすさを重視。 作業比率を色とグラフで示し、数字だけでは見えにくい偏りを直感的に伝えます。

Judge Point

ChronoMeの魅力は、学習用アプリで終わらず「使い続ける理由」まで設計している点です。 企画、実装、データ設計、運用計画、UI表現が同じ目的に向かって接続されています。
