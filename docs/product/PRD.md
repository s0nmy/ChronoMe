# PRD (Product Requirements Document)

一人用タイムカード Webアプリ

## Overview / Problem Statement

多くの個人事業主やフリーランス、学習者は、日々の作業時間を正確に把握することに課題を抱えている。
既存の勤怠管理システムやタスク管理アプリは「チーム向け」「企業向け」に設計されていることが多く、一人で複数の作業を行うユーザーにとってはオーバースペックまたは使いづらいケースが多い。

特に以下の問題が存在する

* 作業を複数同時に進めた場合に、時間配分を柔軟に記録できない
* 個人利用においては、シンプルで素早い操作が求められるが、既存サービスは複雑すぎる
* 日や週の作業量を俯瞰したいが、レポートやカレンダー機能が不足している

このプロジェクトでは「個人が日々の作業時間を簡単に記録・分析できる仕組み」を提供する。

## Goals

* ユーザーは、一日の中で複数の作業にどれだけ時間を使ったかを可視化できる
* ユーザーは、週や月の作業量を振り返り、仕事・学習・休憩のバランスを改善できる
* ユーザーは、シンプルな操作で作業時間を記録でき、習慣として継続できる

## Low-Goals

* グラフやカレンダーを使った可視化により、作業時間を直感的に把握できる
* CSV/JSON形式でデータをエクスポートできる
* 簡易的なタグ付けにより、作業内容を分類できる

## Non-Goals

* チーム単位での勤怠管理
* 従業員向けの承認ワークフロー
* AIによる自動分類や自動入力

## User Stories

### フリーランスのデザイナー（40歳）

* 複数クライアントの案件を掛け持ち  
* 頭の中で作業が混在し、1日の作業時間を振り返りにくい  
* ニーズ：作業内容ごとに簡単に記録し、週単位で振り返りたい

### 学生ユーザー（21歳）

* 学業とアルバイトを並行  
* 勉強時間と労働時間のバランスを見たい  
* ニーズ：毎日の勉強時間をタグで管理し、月単位で成果を確認したい

### 個人開発者（30歳）

* 副業で個人アプリを開発中  
* 日常的に複数作業を切り替える  
* ニーズ：コード作業、調査、設計などを分けて記録し、作業比率をグラフで把握したい

## Proposed Solution / Feature Description

- 認証: サインアップ/ログイン/ログアウト/ログインユーザー取得
- タイムエントリ: 作業開始/終了時刻の記録、編集、削除
- プロジェクト: 作業をプロジェクトに紐づけて管理
- タグ: 作業内容の分類と絞り込み
- レポート: 日次/週次/月次の集計サマリ表示

## (High-Level Tech Consideration)

- フロントエンド: React + TypeScript (Vite), TanStack Query, Tailwind CSS
- バックエンド: Go + Chi, Clean Architecture 構成
- DB: SQLite (初期開発) / PostgreSQL (移行先)
- ORM/マイグレーション: GORM, golang-migrate
- API: REST (JSON), `/api` 配下に集約

## Open Questions / Risks

- レポート集計のパフォーマンスをどのタイミングで最適化するか
- SQLite から PostgreSQL への移行手順と運用負荷
- 認証機構の強化要否 (現状はシンプルなセッション認証)
