# 機能対応表（Web版 vs iOS版）

## 概要

このドキュメントでは、既存のWeb版とiOS版の機能対応を定義します。基本機能はWeb版と同等にし、iOS固有の機能を追加することで、より優れたモバイル体験を提供します。

## 機能対応マトリックス

### 凡例

- ✅ 実装予定（必須）
- 🔄 実装予定（改良あり）
- 📱 iOS固有機能
- ⏳ 将来実装予定
- ❌ 実装予定なし

## データ同期

### オフライン対応

| 操作 | オフライン時 | オンライン復帰時 |
|-----|------------|---------------|
| エントリ作成 | ローカルに保存 | サーバーと同期、競合解決 |
| エントリ編集 | ローカルに保存 | サーバーと同期、競合解決 |
| エントリ削除 | ローカルで削除マーク | サーバーに削除リクエスト |
| プロジェクト/タグ作成 | ローカルに保存 | サーバーと同期 |
| レポート表示 | キャッシュデータ表示 | 最新データ取得 |

### 同期戦略

- **差分同期**: 変更されたデータのみ送信
- **競合解決**: サーバータイムスタンプを優先（Last Write Wins）
- **バックグラウンド同期**: アプリがバックグラウンドの際も定期的に同期

## まとめ

### Phase 1（MVP）で実装する機能

- 全コア機能（認証、タイムエントリ、プロジェクト、タグ、レポート）
- 基本的なウィジェット（小・中サイズ）
- 基本的な通知（リマインダー）
- オフライン対応
- 生体認証

### Phase 2で実装する機能

- 高度なウィジェット（大サイズ、ロック画面）
- ショートカット連携
- iPad最適化
- 証明書ピンニング

### Phase 3で実装する機能

- ダイナミックアイランド対応
- Apple Watch連携
- Handoff対応
- インタラクティブウィジェット

iOS版は、Web版の機能を完全にカバーしつつ、モバイルファーストな体験とiOS固有機能により、より優れたユーザー体験を提供します。
