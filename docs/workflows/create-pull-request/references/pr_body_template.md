# PR 本文テンプレート

`../../../../REVIEW.md` の「PR説明欄の更新」に合わせたテンプレート。Mermaid は必要な変更でのみ使い、この workflow では省略してよい。

## 変更サマリー

何を・なぜ変更したかを 3〜5 行で記述する。影響レイヤーも明記する。

例:

- iOS の初期セットアップを追加し、アプリ起動に必要な基本構成を整備した。
- API 接続先の設定方法を見直し、開発環境ごとの差し替えを安全にした。
- 影響レイヤー: iOS / 設定ファイル

## テスト方法

変更内容に応じて確認すべき手順を列挙する。

- [ ] 単体テスト
- [ ] API エンドポイントの入出力確認
- [ ] 実機またはシミュレータでの手動確認
- [ ] 外部連携の確認

実施していない項目は未チェックのまま残し、必要なら補足を書く。

## 考慮事項

変更に伴うリスクや注意点を列挙する。

- プライバシーへの影響
- セキュリティへの影響
- 後方互換性への影響
- パフォーマンスへの影響

該当しない場合は `該当なし` と明記する。
